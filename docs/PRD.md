# Valley Steward 产品需求文档

版本：0.1（MVP 框架）  
日期：2026-07-13

## 1. 产品概述

Valley Steward 是面向《星露谷物语》PC 玩家的桌面 Mod 管理器。它把 SMAPI 安装、游戏路径定位、Mod 解压、版本更新和文件清理收敛为可检查、可回退的操作，降低新手直接编辑游戏目录的风险。

参考《星露谷物语 Wiki》入门教程，本产品遵循以下事实：

- SMAPI 是大多数 Mod 与游戏之间的加载层；
- SMAPI Mod 应位于游戏目录的 `Mods` 子目录，每个 Mod 通常有自己的目录和 `manifest.json`；
- SMAPI 会递归扫描 `Mods` 的子目录；
- 目录名前加 `.` 可禁用 Mod；
- 更新 Mod 时应覆盖程序文件，但保留 `config.json` 等用户配置；
- 删除 Mod 等价于移除其目录，但应先检查发布页的卸载说明；
- 仅替换 `.xnb` 且没有 `manifest.json` 的旧式 Mod 风险较高，MVP 不自动安装。

参考来源：[Wiki 入门教程](https://zh.stardewvalleywiki.com/%E6%A8%A1%E7%BB%84:%E4%BD%BF%E7%94%A8%E6%8C%87%E5%8D%97/%E5%85%A5%E9%97%A8)

## 2. 目标用户

- 第一次安装 SMAPI 或 Mod 的中文玩家；
- 已有多个 Mod，希望快速检查、禁用、更新和清理的玩家；
- 需要在单人、联机等不同 Mod 组合间切换的进阶玩家（后续版本）。

## 3. 产品目标

1. 首次启动 3 分钟内完成游戏路径与 SMAPI 状态确认。
2. 用户可从本地压缩包或上游站点完成安全安装。
3. 用户能快速判断 Mod 的启用、依赖、兼容与更新状态。
4. 默认不执行不可逆文件操作，更新失败可回退。

## 4. MVP 范围

### 4.1 环境检测

- 自动检查 Steam、GOG、Xbox App 的常见安装路径；支持 Windows、macOS、Linux 的默认路径。
- 通过游戏可执行文件验证目录，而不是只按文件夹名判断。
- 支持手动选择目录，并持久化最近一次有效路径。
- 检测 `StardewModdingAPI` 可执行文件、版本信息与 `Mods` 目录。
- SMAPI 未安装时展示官方安装入口；安装包下载完成后引导用户确认并执行官方安装程序。

验收：路径无效时给出明确原因；重新扫描不会破坏已有配置。

### 4.2 基础 Mod 管理

- 递归扫描 `Mods`，解析 `manifest.json` 的名称、作者、版本、唯一 ID、依赖和更新键。
- 展示启用、禁用、异常、可更新四类状态。
- 支持搜索、状态筛选、排序和刷新。
- 支持打开 Mod 目录、查看清单与配置文件。
- 启用/禁用通过安全重命名实现，并检测目标目录冲突。

验收：单个无效清单不会阻断其他 Mod 扫描；错误项可定位到具体路径。

### 4.3 文件安装与删除

- 支持拖入或选择 `.zip` / `.7z` 文件；先解压到临时目录再检测真正的 Mod 根目录。
- 安装前显示清单、依赖、同名冲突和 XNB 风险。
- 快速删除默认移动到 `.mod-manager-trash`，提供恢复与定期清空。
- 正在运行游戏时禁止覆盖、删除和重命名 Mod。

验收：压缩包中的多 Mod、额外说明目录和数字后缀外层目录可被识别。

### 4.4 Mod 更新

- 根据 `UpdateKeys` 关联上游版本；没有更新键的 Mod 标记为“无法自动检查”。
- 下载到缓存，校验压缩包后在临时目录展开。
- 原子替换程序文件；保留 `config.json`、用户声明的配置和回滚快照。
- 支持单个更新与批量更新，失败项互不影响。

验收：更新失败后原 Mod 仍可启动；版本变化和文件操作写入本地日志。

### 4.5 快速下载与上游站点

- 首批适配 Nexus Mods、ModDrop、GitHub Releases 与 SMAPI 兼容性目录。
- 下载中心默认直接聚合并展示可浏览的 Mod 列表，来源作为筛选条件；浏览阶段不要求用户先连接账号。
- 统一展示来源、版本、发布时间、游戏版本、SMAPI 版本、依赖与许可证。
- Nexus 使用用户自有 API Key 或浏览器授权，不绕过登录、限速或作者分发规则。
- API Key 由用户在设置中填写，验证后进入系统凭据库，不写入前端存储、配置文件或日志。
- 不抓取 Wiki 明确警告的转载站，不镜像受限制文件。
- 无 API 的站点仅打开官方发布页，由用户完成下载后自动接管本地文件。

验收：任何下载都能追溯来源 URL 和版本；认证信息进入系统安全存储，不写入日志。

### 4.6 一键启动

- 顶部主按钮默认通过 SMAPI 启动游戏；可切换为原版游戏。
- 提供“记住上次选择”，启用后主按钮在下次启动应用时沿用最近一次启动方式。
- 支持向 SMAPI 传递 `--mods-path`，快速启动指定 Mod 配置目录。
- 支持受控的附加参数列表，Rust 使用进程 API 逐项传递，不拼接 Shell 命令。
- 启动前检查游戏路径、目标可执行文件和 Mods 目录边界，避免执行任意路径程序。
- 启动成功后持续监测由管理器创建的游戏进程，在界面显示运行、已退出或未运行状态。
- 游戏运行时提供一键关闭和一键重启；重启必须复用被监测进程的启动模式、Mods 路径和附加参数。
- 关闭仅终止管理器创建并持有的 Windows Job Object 或 Unix 进程组，不按模糊进程名查找或批量结束其他进程。
- 从 Steam、终端或其他工具外部启动的游戏不纳入当前会话的关闭和重启范围。
- 后续支持保存“单人”“联机”“测试”等启动预设，并展示最近一次启动结果。

验收：选择 SMAPI 时必须检测到其可执行文件；路径或参数不合法时不创建进程并给出原因；启动后状态在短轮询周期内更新；关闭后被跟踪的进程树退出；重启后的启动请求与本次记录一致。

## 5. 核心流程

### 首次启动

自动探测游戏 -> 用户确认路径 -> 检测 SMAPI -> 安装或跳过 -> 扫描 Mods -> 进入工作台。

### 安装 Mod

选择来源/本地包 -> 下载到缓存 -> 解压到临时目录 -> 解析清单与依赖 -> 用户确认 -> 原子移动到 `Mods` -> 重新扫描。

### 更新 Mod

检查更新 -> 展示变更与依赖 -> 创建回滚快照 -> 下载并校验 -> 覆盖且保留配置 -> 重新扫描 -> 成功后清理旧快照。

### 删除 Mod

检查游戏进程与卸载提示 -> 移入回收区 -> 刷新列表 -> 支持恢复或永久清空。

### 启动与进程控制

校验启动请求 -> 创建并记录游戏进程 -> 持续刷新运行状态 -> 用户关闭或游戏自然退出；选择重启时，先结束被跟踪的进程树，再使用已记录的启动请求重新创建进程。

## 6. 信息架构

- 概览：环境状态、待处理事项、最近活动；
- 我的 Mod：列表、筛选、启停、更新、删除、打开目录；
- 下载中心：搜索上游、下载队列、本地包安装；
- SMAPI：检测、版本、安装/更新、日志入口；
- 设置：游戏路径、下载目录、更新与安全策略、上游账号。
- 顶部启动器：SMAPI、原版游戏与指定 Mod 配置集。

## 7. 数据模型

`GameInstallation`：平台、商店、游戏路径、版本、可执行文件。  
`SmapiStatus`：是否安装、版本、可执行文件、日志目录。  
`InstalledMod`：唯一 ID、名称、作者、版本、路径、启用状态、依赖、更新键、健康状态。  
`RemoteMod`：来源、来源 ID、版本、下载 URL、主页、许可证。  
`Operation`：类型、目标、状态、进度、错误、时间、回滚位置。

`GameProcessStatus`：运行状态、被跟踪 PID、启动目标与启动时间。

`LaunchRequest`：游戏路径、启动目标、Mods 路径和附加参数；进程存活期间保存在内存中供安全重启复用。

## 8. 非功能需求

- 安全：Rust 后端规范化所有路径，拒绝压缩包路径穿越和 `Mods` 外写入。
- 性能：1,000 个 Mod 的本地扫描目标小于 2 秒；扫描和下载不阻塞 UI。
- 隐私：除上游 API 请求外不上传本地 Mod 列表、游戏路径或日志。
- 可访问性：键盘可操作、状态不只依赖颜色、支持降低动态效果。
- 可观测性：本地结构化日志默认脱敏路径和密钥。
- 进程安全：关闭和重启不接受前端提交的 PID，只操作后端持有的 Job/进程组，也不使用模糊进程名终止游戏。

## 9. 版本路线

- `0.1`：路径探测、扫描、SMAPI 检测、启停、可恢复删除、UI 框架。
- `0.2`：本地压缩包安全安装、配置保护、操作日志与回滚。
- `0.3`：Nexus/GitHub 上游适配、更新检查与下载队列。
- `0.4`：SMAPI 官方安装流程、依赖解析、批量更新。
- `1.0`：多配置集、冲突提示、存档备份与稳定迁移。

## 10. 暂不实现

- 自动安装 XNB 替换型 Mod；
- 绕过上游登录、会员限速或下载限制；
- 自动修改存档；
- 未经确认永久删除文件；
- 在主机与 iOS 平台管理 Mod。
